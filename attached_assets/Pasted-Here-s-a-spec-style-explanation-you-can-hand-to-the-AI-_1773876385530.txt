Here’s a spec-style explanation you can hand to the AI so it can build these features in one pass. It’s written assuming React + typical SPA patterns, but is framework-agnostic.Emoji reactionsImplement Discord-style reactions on each message: a compact row of “reaction pills” showing emoji and counts.�Behavior:Each message can have zero or more reactions.A reaction pill shows:The emoji (Unicode or custom image).A numeric count.A subtle highlight state if the current user has reacted with that emoji.Clicking a reaction pill:If the user has reacted with that emoji: remove their reaction and decrement the count.Otherwise: add their reaction and increment the count.Hovering a message shows a minimal reaction affordance (e.g. a “+” or smiley icon) near the top-right of the message row; clicking it opens the emoji picker scoped to “add a reaction” for that message.�Large numbers (1000+) can be abbreviated (e.g. 1.2K) to keep pills small.Data model:Each message has a reactions array:emojiId (custom emoji id or Unicode codepoint string).count (total).reactedByCurrentUser (bool).On the backend, store per-user reaction rows (messageId, userId, emojiId); derive counts for the UI.Layout / styling:Reaction pills sit under the message bubble, aligned left, in a row with 4–8px gap.Pills use a subtle background (e.g. gray-800 in dark mode) with rounded-full border radius.The “reacted by me” state can use a slightly stronger background or border to be visually distinct but not loud.For many reactions, wrap to second line; do not overflow horizontally.Limit visible reaction types (e.g. first 8 by count), with a “+N” overflow pill if there are more.Message threads / repliesImplement per-message threads that open in a right-hand sidebar, like Discord’s “Thread” or Mattermost’s thread sidebar.�Core behavior:Every top-level message in a channel can act as a thread root.For any message:Provide a “Reply in thread” action in the message hover toolbar.Clicking it:Opens a thread sidebar anchored to that root message.Focuses a thread-specific composer at the bottom of that sidebar.Replies are not shown inline in the main channel list (or only as a very compact indicator) to reduce clutter in busy channels.�Main channel view:Each message can optionally show:A small “X replies” link beneath it if it has a thread.Last reply timestamp and last replier name, e.g. “5 replies -  Last reply by Alice 2m ago”.Clicking this “X replies” opens the thread sidebar for that message.Thread sidebar:Appears on the right, pushing or overlaying the main content depending on viewport width.Header includes:“Thread” label.Snippet of the root message (author, truncated content).A close button.Body shows:The root message at the top (in slightly dimmer style, or with “Original message” label).All replies in chronological order, like a mini-channel.Bottom shows:A thread-specific composer box; messages sent here are linked to the thread root.Data model:Each message has:Optional threadId.Each thread has:id, rootMessageId, channelId.replyCount, lastReplyAt, lastReplier.Replies are just messages whose threadId points to the thread; backend enforces that they live in the same parent channel.Navigation:Thread state is part of the URL if possible (/server/:id/channel/:id/thread/:threadId) so users can deep-link.Closing the sidebar pops you back to the plain channel route.@mentionsImplement mentions that visually highlight references to users, roles, and “@everyone/@here”, and drive notifications.�Parsing:In the composer, when the user types @, show an autocomplete popover:First users in the current channel, then roles, then special values (@everyone, @here).On selection, insert the mention in a structured way (e.g. @Danny but store as { type: 'user', id: '...' } in the message payload).On the backend, store mentions as structured metadata attached to each message:mentions: { users: [...], roles: [...], everyone: bool, here: bool }.Rendering:Mentions render inline as pill-like tokens:Slight background highlight and bold text.Different colors per type (user vs role vs @everyone).Clicking a user mention opens that user’s profile card (see “User profiles”).Hover state can show a tooltip with more info (e.g. full username#discriminator).Notifications:When a message includes a user mention:Generate a notification event for that user if they have not muted the channel.When a message mentions a role or @everyone:Generate notifications for all members who:Have permission to see the channel.Have not disabled @role/@everyone pings for that server or channel.�Channels with unread mentions show:A mention badge (e.g. red dot with count) next to the channel name.Users can configure notification levels per server and per channel:All messages.Mentions only.Nothing / mute.�Server & channel settingsImplement editable server-level and channel-level settings, with appropriate mod/admin-only access.�Server settings:Access:A server dropdown at the top of the left sidebar shows server name, icon.“Server Settings” is visible only to users with “Manage Server” permission.Settings screen includes:Change server name (text input).Change server icon (image upload with crop).Optional: default notification level, default AFK channel, etc.Persist changes immediately after “Save”; optimistic UI is fine with rollback on error.Channel settings:For each channel in the server’s channel list:Right-click or a “gear” icon opens “Channel Settings” for users with “Manage Channels”.Channel settings include:Rename channel (name input).Delete channel (danger zone with confirmation dialog).Channel topic (short description displayed at the top of the message list, under the channel name).Optional: per-channel notification override, permissions, etc.Renaming a channel removes the channel from the list and navigates the user to a safe fallback channel.User profilesImplement lightweight profile cards that appear when clicking a username or avatar.�Invocation:Anywhere a username or avatar appears (message header, member list, mentions):Clicking it opens a floating profile card near the cursor.For mobile, tapping opens a bottom sheet instead.Profile card contents:Top section:Avatar (large).Display name and unique handle/id.Middle section:Join date for this server (e.g. “Joined May 2024”).Roles, displayed as colored tags/badges.Bottom section:Quick
