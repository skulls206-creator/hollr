This stays inside your Hollr right‑hand column; Hollr just renders <FoldrPanel hollrUser={currentUser} activeChat={currentChat} />.

Props and data model
Key props to pass from Hollr:

hollrUser: { id, username, avatarUrl }

activeChat: { id, type: 'dm'|'channel', name }

Core Foldr entities (front‑end models, backed by your API):

Workspace – one per chat or user:

{ id, name, scopeType: 'chat'|'user'|'global', scopeId }

Folder:

{ id, name, parentId, workspaceId, path }

FileItem:

{ id, name, size, mimeType, cid, ipfsGatewayUrl, folderId, workspaceId, owner, createdAt, updatedAt, access, isStarred, isDeleted }

Share:

{ id, fileId, targetType: 'user'|'channel'|'link', targetId, permission: 'view'|'download'|'edit', expiresAt }

Upload flow
You want a consistent path for drag‑and‑drop, toolbar, and “Chat attachments” uploads.

Implementation sketch:

<UploadDropzone> wraps the whole <FoldrMain> area; on drop, call handleUpload(files, { workspaceId, folderId, origin: 'foldr' | 'chat' }).

handleUpload should:

Immediately show local “pending files” rows with status “Uploading…”.

POST to your backend, which uses @lighthouse-web3/sdk or API to upload and returns cid, ipfsGatewayUrl.

Replace pending entries with final FileItems.

ShareModal
Pulled up from details drawer or row action.

Fields:

“Who can access?” – radio: Only me, People / channels I choose, Anyone with link.

For “People / channels I choose”:

Hollr search input (users + channels).

Chips showing selected principals.

Per‑principal permission: view or download.

Optional: “Link expires on” date picker, “Add password”.

Backend endpoints:

GET /foldr/files/{id}/shares

POST /foldr/files/{id}/shares

DELETE /foldr/shares/{id}.

Chat integration hooks
From Foldr back into Hollr, you just need callbacks:

onSendToChat(file: FileItem) prop on FoldrPanel that Hollr implements as “insert message with Foldr attachment”.

When a message contains a Foldr attachment, clicking it opens the Foldr panel focussed on that file’s workspace/folder and selects it.

Internally, Foldr should tag files with origin metadata:

{ origin: 'chat' | 'upload', originChatId?: string, originMessageId?: string }

So “Chat Attachments” view is a simple filter on origin='chat' and originChatId=activeChat.id.
